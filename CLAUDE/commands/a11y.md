# /a11y — Accessibility Review

When Rob runs /a11y [component], audit it for accessibility. TM Stats must work for golfers of all ages — including older club members who may have visual or motor impairments.

## Why This Matters for TM Stats

The target market includes 50–70 year old club golfers. Poor accessibility = lost subscribers. England Golf would also require accessibility compliance for any partnership.

---

## The 5 Checks

### Check 1 — Tap Targets (Mobile Critical)
- Every interactive element must be at least 44×44px
- YES/NO toggles: full width, minimum 52px height
- Score steppers: minimum 52×52px buttons
- "Next Hole" button: minimum 56px height, full width on mobile
- Check any icon-only buttons — do they have a visible label or tooltip?

### Check 2 — Colour Contrast
- Text on dark backgrounds must meet WCAG AA (4.5:1 ratio minimum)
- Check: white text on #CC2222 red — is it readable? (it is, but verify)
- Check: #9A9DB0 secondary text on #0F1117 background — verify ratio
- SG positive green (#22C55E) on dark background — verify
- Never rely on colour alone to convey meaning — SG positive/negative must also show +/- sign and label
- Colourblind users: SG bars should have +/- labels, not just green/red

### Check 3 — Screen Reader Support
- All images and icons must have `alt` text or `aria-label`
- YES/NO toggles must have `role="radiogroup"` with accessible labels
- The hole progress bar needs `aria-label="Hole 7 of 18"`
- SG numbers need context: not just "-1.4" but `aria-label="Approach shots, minus 1.4 versus scratch"`
- Form inputs need visible labels — not just placeholder text (placeholder disappears when typing)

### Check 4 — Focus & Keyboard Navigation
- Tab order must follow the logical flow through the round entry form
- Focus ring must be visible — use `outline: 2px solid #CC2222` on focus
- No keyboard trap — user must always be able to tab out of any component
- Score stepper must be operable with keyboard arrow keys, not just mouse/touch

### Check 5 — Error States
- Error messages must be announced to screen readers (`role="alert"`)
- Errors must appear near the field they relate to — not in a distant toast
- Never use colour alone to indicate an error — use an icon and text too
- If the round save fails, the error must be prominent and explain what to do

## Output Format

### ✅ Passes
What's accessible already.

### 🔴 Must Fix — Will Block Real Users
Issues that would prevent a significant portion of users from using the feature.

### 🟡 Should Fix
Issues that degrade the experience for some users.

### 📋 Accessibility Score
[X/5 checks passed] — ACCESSIBLE / NEEDS WORK / FAILING
